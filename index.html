About the Visualization

This, d3.js (v4) final project, part of CS 498 Data Visualization course in UUIC, uses a hybrid approach of an Interactive Slideshow in order to narrate the visual story of Financial Market dataset from FINVIZ.COM. The Interactive Slideshow takes the user throughout the July-09 2018 single trading day, zooming from high-to-low level insights (more details to follow).

The Interactive Slideshow is built using scenes with annotations, while having parameters connected with the related triggers. In order to preserve visual consistency along the Slideshow, strict and constant templates are being used. More elaboration and details related to all the mentioned elements and of course the visualizations to follow later.

Recommended View Settings

As the visualizations use a preset static canvas of 1024x768 pixels, the viewing size of the Interactive Slideshow is at least 1280x800 pixels (laptop screen or a monitor are advised) in order to accommodate the visualizations, surrounding text and scene elements. The system was comprehensively tested on Windows 10 (Chrome, Firefox) and MacOS (Chrome, Safari).

ELEMENTS OF NARRATION

  1. Scenes

    2. Scenes are the slides that make up the Slideshow; these slides can be easily navigated using the guiding arrows in the corner. The use of CSS and FullPage.js allows for a consistent template look for the Interactive Slideshow visualization; each page is transitioned using the same method, while the main concepts are kept the same (e.g. font family). As noted, the smart navigation control between pages allows users to easily and smoothly navigate between the slides.

  2. Annotations

    3. Annotions are the miscellaneous labels, tooltips and legends; those display the key attributes from our Dataset (more details on the latter to follow) - such as Company, Country, Market Capitalization and Sector. The tooltips are triggered when the user interactively moves the mouse over the visualization elements and this invokes the display parameters for the each relevant item and hence it becomes visible to the user.

    In general, the annotations that are being used in the visualizations all have a similar process of using triggers to change the hidden parameter of the annotation. For example, the INTERACTIVE ANALYSIS OF THE WORLD chart has an initial state of the hidden parameter which controls the display of the annotations set to false value. As the user triggers changes to the chart parameters, it also changes this hidden parameter to true value. In its' turn, the annotations disappear until the user resets chart to default - meaning zooming out back to the chart default visualization parameters.

  3. Parameters

    4. Parameters are the main characteristic attributes of the Slideshow, in our case they are Company, Sector, Market Capitalization and Country. There are additional parameters that are listed in the Dataset, yet those are relatively minor and appear as details in the visualization charts, rather being considered as one of main parameters for the visualizations.

  4. Triggers

    5. Triggers are easy discoverable by the end-user Zoom and Brush filter, and buttons with related control details. Triggers are being used in order to invoke updates in the visualizations based on the new parameters set by the end-user.

    Both parameters and triggers are intensively used in all of the visualizations, with an emphasis on the INTERACTIVE ANALYSIS OF THE WORLD and MAJOR COMPANIES INTERACTIVE INSIGHTS charts. For example, in INTERACTIVE ANALYSIS OF THE WORLD chart parameters for the (x, y) coordinates of each Bubble are set to an initial central position for the All Companies Visualization. As the user selects various grouping options (Sector, Country and Market Capitalization) it triggers the change of the Bubble’s (x, y) coordinates parameter to their respective groupings. In the MAJOR COMPANIES INTERACTIVE INSIGHTS chart, every mouse click on a region is a trigger for the path and arc parameters. By clicking within a region, the user zooms into the data information underneath, while clicking the central circle will trigger the parameters to return to its values a level up in the hierarchy.

DATASET DETAILS

    Containing nearly 10,000 traded companies, The FINVIZ.COM is considered as one of the major financial data providers in the world. It constantly (real-time) searches through large amount of stock data and collects them in a single source that allows end-user various monitoring, querying and analysis capabilities.

    Relevant stock exchanges (where the company is listed) - all stocks listed on: National Association of Securities Dealers Automated Quotation (NASDAQ), New York Stock Exchange (NYSE) and American Stock Exchange (AMEX). Snapshot data: July 09, 2018 (single day performance). After export, the data was cleaned and preprocessed using common data curation activities. In addition to Company and relevant trading Ticker, following attributes were extracted:

  1. Sector

    2. Companies are divided into several groups - sectors - according to their business activities.

  2. Industry

    3. Companies in a common sector are further divided by products and services into smaller groups - industries.

  3. Country

    4. The geographic location of a company (listed on U.S. markets).

  4. Market Capitalization (Cap)

    5. The total dollar value of all of a company's outstanding shares. Market capitalization is a an essential measure of corporate size.

  5. P/E

    6. A popular valuation ratio of a company's current share price compared to its per-share earnings (trailing twelve months). Low P/E value indicates a stock is relatively cheap compared to its earnings.

  6. Price

    7. The current stock price or the close price during the last trading session. In our case it is last trading session.

  7. Change

    8. The difference, in %, between current close and previous close price.

  8. Volume

    9. Total number of shares traded for a given stock today, or during the last trading session.

WHAT IS NEXT?

This was quite a humble attempt to present various visualizations of financial data in an easy interactive manner to the end user. Obviously, these financial insights are just a beginning!

On top of various visualizations and possible extensions, we haven't even visualized a very significant dimension for the stocks analysis - the time; an obvious next step would be finding trends in stock prices and volumes - how those have changed over the time (years, months, weeks, days and within the day). For the latter process, a more automated data flow should be setup via a SW API (comparing to one time data export as was done for this project...).
